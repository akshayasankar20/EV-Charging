╔════════════════════════════════════════════════════════════════════════════════╗
║                            DATA FLOW EXAMPLE                                   ║
╚════════════════════════════════════════════════════════════════════════════════╝

1️⃣  USER ACTION
   User opens frontend → "Find Best Station" button

2️⃣  FRONTEND REQUEST (React)
   POST /api/recommend { user_location: {lat: 40.71, lng: -74} }

3️⃣  BACKEND PROCESSING (Node.js)
   ├─ Retrieve all stations from in-memory store
   ├─ For each station calculate:
   │  ├─ Waiting Time = (Queue / Chargers) × Avg_Charge_Time
   │  ├─ Availability = Available / Total
   │  └─ Load Factor = Current_Draw / Capacity
   ├─ Calculate optimization score
   └─ Return best station + alternatives

4️⃣  EDGE SIMULATION (Real-time)
   ├─ Update queue positions
   ├─ Simulate user completions
   ├─ Adjust power draw
   └─ Update availability every 5 seconds

5️⃣  FRONTEND DISPLAYS
   ├─ Recommended station (highlighted)
   ├─ Waiting time estimate
   ├─ Available chargers
   ├─ Grid load status
   └─ "Book Now" button ready

6️⃣  USER BOOKS
   POST /api/book { station_id, user_id, vehicle_type }
   
7️⃣  BOOKING CONFIRMED
   ├─ Backend adds to queue
   ├─ Returns booking ID
   ├─ Frontend shows confirmation
   └─ Queue position tracked


╔════════════════════════════════════════════════════════════════════════════════╗
║                        OPTIMIZATION ALGORITHM                                  ║
╚════════════════════════════════════════════════════════════════════════════════╝

For each station:

    Waiting_Time = (Queue_Length / Total_Chargers) × Avg_Charging_Time
    
    Availability_Score = Available_Chargers / Total_Chargers
    
    Load_Factor = Current_Power_Draw / Power_Capacity
    
    Base_Score = (100 - Waiting_Time) × 0.40
               + Availability_Score × 100 × 0.30
               + (1 - Load_Factor) × 100 × 0.30
    
    IF Load_Factor > 0.85:
        Final_Score = Base_Score - 50  (Heavy Penalty)
    ELSE:
        Final_Score = Base_Score

RESULT: Station with HIGHEST Final_Score is recommended ✓
